# 医疗医护后台管理系统 (Vue3 + TS + Element-Plus)

项目简介
本项目是一款基于 Vue3 生态栈开发的**医疗医护后台管理系统**。主要面向医护机构提供订单管理、医护员调度、用户权限控制等核心功能。

技术栈
框架: Vue 3 (Composition API)
构建工具: Vite
状态管理: Pinia (或 Vuex，请根据你实际情况改)
UI 组件库: Element Plus
网络请求: Axios (实现二次封装与拦截)
路由: Vue Router (包含动态权限路由逻辑)

核心功能与技术亮点
Axios 二次封装: 实现了全局请求/响应拦截器。统一处理 Token 注入、业务错误码拦截（如 401, 500）以及全局 Loading/Message 提示。
权限管理 (RBAC): 实现了基于角色的访问控制，支持根据用户角色动态挂载路由，确保系统安全性。
业务逻辑:
    订单全生命周期管理: 涵盖待支付、接单、服务中、已完成等状态流转。
    医护员管理: 包含医护员入驻、审核及状态监控。
工程化规范: 项目遵循 ESLint 代码规范，采用模块化开发模式，具备良好的可维护性。

项目运行
1. 安装依赖: `npm install`
2. 启动项目: `npm run dev`
3. 项目打包: `npm run build`

开发心得：

工程化思想的建立：通过本项目，我深刻理解了 Axios 二次封装 的必要性。在开发初期，我发现每个页面手动处理 Token 和错误码非常低效。通过引入请求拦截器，我实现了全站身份信息的自动注入；利用响应拦截器，实现了业务错误（如 401/500）的集中捕获，使前端逻辑更加纯粹。

Vue3 响应式原理实践：在使用 Composition API 开发医护员列表时，我通过 ref 与 reactive 的灵活配合，解决了深层数据更新不触发视图渲染的问题，并对组件的卸载阶段进行了处理，防止了 Echarts 实例导致的内存泄漏。

项目部署与规范：初次尝试将项目部署并维护在 GitHub 上，学习了 Git 的分支管理和 commit 规范，意识到了良好的代码注释和目录结构对团队协作的重要性。
